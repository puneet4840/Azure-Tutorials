# Azure Front Door Real World Example

Azure Front Door ka request flow ek real-world production example ke saath samjhenge.

<br>

### Real World Scenario Setup

Maan lo ek company hai: **ShopSphere (Global E-Commerce Platform)**.

Iski website hai:
```
https://www.shopsphere.com
```

Iska infrastructure Azure par deploy hai.

Architecture:
```
Users (Global)
       │
       ▼
Azure Front Door
       │
       ▼
Backend Regions
 ├─ East US
 ├─ West Europe
 └─ Southeast Asia
```

Har region mein running services:
```
Azure Load Balancer
App Gateway
VM Scale Sets
Database
```

Backend pool example:
```
EastUS
   VM1
   VM2

WestEurope
   VM3
   VM4

SoutheastAsia
   VM5
   VM6
```

Azure Front Door ka kaam:
```
User request ko nearest edge location par receive karna
Best backend region select karna
Request forward karna
Response return karna
```

<br>

### Step 1 — User Website Open karta hai

Example user:
```
Location : Delhi India
Device : Laptop
Browser : Chrome
```

User browser mein type karta hai:
```
https://www.shopsphere.com
```

Ab ek complete internet workflow start hota hai.

<br>

### Step 2 — DNS Resolution

Browser ko pehle pata karna hota hai:
```
shopsphere.com ka IP kya hai?
```

Browser DNS resolver ko query bhejta hai.
```
User Browser
     │
     ▼
ISP DNS Resolver
```

DNS record:
```
www.shopsphere.com
CNAME
shopsphere.azurefd.net
```

Ab DNS resolver query karta hai:
```
shopsphere.azurefd.net
```

Azure DNS response deta hai:
```
Anycast IP
20.42.x.x
```

Ab browser ke paas IP aa gaya.

<br>

### Step 3 — Anycast Routing

Ye IP ek single server ka IP nahi hota.

Ye Azure Front Door ka Anycast IP hota hai.

Anycast ka matlab:
```
Same IP world ke sabhi POP locations par available hota hai.
```

Example Azure POP locations:
```
Mumbai
Singapore
Tokyo
Amsterdam
London
New York
```

Internet routers BGP protocol use karke decide karte hain:
```
closest location
```

Kyuki user Delhi mein hai.

Routing hoga:
```
User
 │
Delhi ISP
 │
Internet Backbone
 │
Mumbai Azure Edge POP
```
Ab request Mumbai edge location par land karti hai.

<br>

### Step 4 — Azure Edge POP par Request Arrival

Edge POP kya hota hai?

Ek mini datacenter jo internet edge par hota hai.

POP ke andar components:
```
Edge routers
Layer 7 proxies
TLS termination servers
WAF engines
Cache servers
Routing engines
```

Infrastructure flow:
```
Internet
   │
Edge Router
   │
Edge Load Balancer
   │
Front Door Proxy Cluster
```

Ab request proxy server tak pahunchti hai.

<br>

### Step 5 — TLS Handshake (HTTPS Connection)

Kyuki website HTTPS hai.

Browser aur edge server ke beech TLS handshake hota hai.

Handshake steps:

1-Client Hello

Browser bolta hai:
```
Supported TLS versions
Supported cipher suites
Random key
```

2-Server Hello

Edge server reply karta hai:
```
Chosen cipher
SSL certificate
Random key
```

Certificate example:
```
www.shopsphere.com
Issued by DigiCert
```

Browser certificate verify karta hai.

Agar valid hai to encryption start hoti hai.

Ab connection secure ho gaya.

<br>

### Step 6 — Web Application Firewall Inspection

Azure Front Door mein WAF enabled hai.

Ab request security engine se pass hoti hai.

Example request:
```
GET /products
```

WAF check karta hai:
```
SQL injection
XSS
Bad bots
Malicious payload
```

Example malicious request:
```
/login?username=admin' OR '1'='1
```

WAF detect karega:
```
SQL Injection attack
```

Aur request block ho jayegi.

Agar request normal hai to next step.

<br>

### Step 7 — Routing Engine Decision

Ab Front Door decide karta hai:
```
kaunsa backend region best hai
```

Routing factors:
```
User location
Backend latency
Backend health
Routing policy
```

Backend latency measurement:
```
Mumbai → SoutheastAsia = 40ms
Mumbai → WestEurope = 120ms
Mumbai → EastUS = 200ms
```

Best backend:
```
Southeast Asia
```

Front Door routing engine choose karta hai:
```
SoutheastAsia backend pool
```

<br>

### Step 8 — Backend Health Check Verification

Routing se pehle Front Door check karta hai:
```
backend healthy hai ya nahi
```
Health probes continuously run ho rahe hote hain.

Example:
```
GET /healthcheck
```

Response:
```
HTTP 200 OK
```
Agar backend healthy hai to routing continue hoti hai.

Agar unhealthy hai:
```
fallback region
```

Example:
```
SoutheastAsia down
→ WestEurope select
```

<br>

### Step 9 — Microsoft Private Infrastructure Journey

Ab request backend region tak jayegi.

Important point:

Azure request ko public internet par nahi bhejta.

Balki Microsoft ke private optical fiber cable infrastructure se hote hue destination region tak pahunchta hai.

Instead use karta hai:
```
Microsoft Global Private Network
```

Infrastructure:
```
200,000+ miles fiber cables
Private routers
Subsea cables
```

Journey:
```
Mumbai Edge POP
       │
Microsoft Backbone Router
       │
Singapore Backbone
       │
Azure SoutheastAsia Datacenter
```
Ye network extremely fast hota hai.

Reason:
```
No internet congestion
High bandwidth
Optimized routing
```

<br>

### Step 10 — Azure Region Infrastructure

Ab request Azure region mein pahunch chuki hai.

Example:
```
Southeast Asia region
```

Region architecture:
```
Regional router
      │
Azure Load Balancer
      │
Application Gateway
      │
VM Scale Set
```

Application Gateway ka kaam:
```
Layer 7 routing
SSL termination (optional)
Path based routing
```

Example request:
```
GET /products
```

Application Gateway forward karega:
```
VM5
```

<br>

### Step 11 — Backend Application Processing

Ab VM5 request receive karta hai.

VM par running hai:
```
Nginx
NodeJS API
Product microservice
```

Processing steps:
```
User request receive
Product database query
Response generate
```

Database query:
```
SELECT * FROM products WHERE category='electronics'
```

Response generate hota hai.

Example response:
```
HTTP 200
JSON data
```

<br>

**Step 12 — Response Back Journey**:

Response wapas same path follow karta hai.

Journey:
```
VM5
   │
Application Gateway
   │
Azure Regional Router
   │
Microsoft Private Backbone
   │
Mumbai Edge POP
   │
User Browser
```

<br>

### Step 13 — Edge Caching

Agar content cacheable hai.

Example:
```
/images/product1.jpg
```

Front Door edge cache kar sakta hai.

Cache storage:
```
Edge POP SSD storage
```

Next user request:
```
Delhi user
```

Request flow:
```
User → Mumbai POP
```

Front Door check karega:
```
Cache available
```

Aur direct serve karega.

Backend hit nahi hoga.

Benefits:
```
Fast response
Lower backend load
Lower cost
```

<br>

# Complete DNS Journey (Browser to Traffic Manager)

Ab hum dekehnge ki Browser se DNS query kaise travel karti hai aur Traffic Manager tak kaise pahuchti hai.

<br>

Hum example use karenge:
```
www.globalshop.com
```
Aur assume karte hain ki is domain ko Azure Traffic Manager ke saath configure kiya gaya hai.

<br>
<br>

### Complete DNS Journey (Browser → Traffic Manager)

High level flow kuch aisa hota hai:
```
User Browser
      |
Local DNS Cache
      |
Operating System Resolver
      |
ISP DNS Resolver
      |
Root DNS Server
      |
TLD DNS Server (.com)
      |
Authoritative DNS Server
      |
Traffic Manager DNS
      |
Best Endpoint Returned
```

Ab isko deep level par samajhte hain.

<br>
<br>

### Step 1 — User Browser mein Domain Enter karta hai

User browser open karta hai aur type karta hai:
```
www.globalshop.com
```

Browser ko nahi pata hota:
```
www.globalshop.com ka IP kya hai
```

Lekin internet par communication IP address se hota hai, domain name se nahi.

Example:
```
Server IP = 52.174.x.x
```

Isliye browser ko domain name ko IP address mein convert karna padta hai.

Ye kaam karta hai:
- DNS (Domain Name System).

<br>
<br>

### Step 2 — Browser Local Cache Check karta hai

Sabse pehle browser check karta hai:
```
Browser DNS Cache
```

Example:
```
Chrome DNS cache
```

Agar domain recently visit kiya gaya hai aur cache expire nahi hua hai:
```
Browser → directly IP use kar lega
```

Agar cache nahi hai:
```
Next step → Operating System DNS resolver
```

<br>
<br>

### Step 3 — Operating System DNS Resolver

Ab query OS ke DNS resolver ko jati hai.

Example:
```
Windows DNS Resolver
```

Ye check karta hai:

System DNS Cache

Location:
```
OS DNS cache memory
```

Agar yahan record mil jata hai:
```
DNS resolved
```
Aur DNS record browser ko mil jata hai.

Agar nahi milta:
```
Query → ISP DNS resolver
```

<br>
<br>

### Step 4 — Request ISP DNS Resolver tak jati hai

Ab request jati hai:
- Recursive DNS Resolver

Ye usually hota hai:
```
ISP DNS Server
```

Examples:
```
Jio DNS
Airtel DNS
Google DNS (8.8.8.8)
Cloudflare DNS (1.1.1.1)
```
Is resolver ka kaam hota hai:
```
Complete DNS lookup perform karna
```

<br>
<br>

### Step 5 — Root DNS Server

Recursive resolver ko bhi nahi pata hota:
```
www.globalshop.com ka IP
```

To wo query karta hai:
```
Root DNS Servers
```
Root DNS servers internet ke top level DNS servers hote hain.

Example:
```
a.root-servers.net
b.root-servers.net
```

Root server IP nahi deta.

Wo sirf direction deta hai.

Example response:
```
Mujhe nahi pata
Lekin .com TLD server ko pata hoga
```

<br>
<br>

### Step 6 — TLD DNS Server (.com)

Ab resolver query karta hai:
```
.com TLD DNS Server
```

TLD ka matlab:
```
Top Level Domain
```

Examples:
```
.com
.org
.net
.in
```

Query:
```
www.globalshop.com
```

TLD server response deta hai:
```
globalshop.com ka authoritative DNS server ye hai
```

Example:
```
ns1.azure-dns.com
```

<br>
<br>

### Step 7 — Authoritative DNS Server

Ab recursive resolver query karta hai:
```
Authoritative DNS Server
```

Ye server domain ka actual DNS record store karta hai.

Example:
```
Azure DNS
Route53
Cloudflare DNS
```

Query:
```
www.globalshop.com
```

DNS record:
```
www.globalshop.com
     CNAME
globalshop.trafficmanager.net
```

Matlab:
```
Website Traffic Manager use kar rahi hai
```

Ab resolver ko next query karni padegi.

<br>
<br>

### Step 8 — Query Traffic Manager DNS par jati hai

Ab recursive resolver query karta hai:
```
globalshop.trafficmanager.net
```

Ye domain Azure Traffic Manager ka managed DNS domain hai.

Example Traffic Manager domain:
```
myapp.trafficmanager.net
```

Ab request pahuchti hai:
- Azure Traffic Manager DNS Infrastructure

Azure ke paas world wide distributed DNS nodes hote hain.

Example:
```
USA
Europe
Asia
Australia
```

User India se request karta hai.

DNS query pahuchti hai nearest node par.

Example:
```
Singapore Traffic Manager node
```

<br>
<br>

### Step 9 — Traffic Manager Routing Engine

Ab Traffic Manager ka internal system activate hota hai.

Traffic Manager check karta hai:
```
1 Routing policy
2 Endpoint health
3 User location
4 Latency mapping
```

Example endpoints:
```
Asia Server
Europe Server
US Server
```

Routing method:
```
Performance routing
```

Latecy Table:
| User Location | Asia | Europe | US    |
| ------------- | ---- | ------ | ----- |
| India         | 40ms | 150ms  | 230ms |

Best endpoint:
```
Asia Server
```

<br>
<br>

### Step 10 — Traffic Manager DNS Response

Traffic Manager DNS response bhejta hai:

Example:
```
asia.globalshop.com
```

ya directly:
```
13.107.x.x
```

DNS response mein TTL bhi hota hai.

Example:
```
TTL = 30 seconds
```

Matlab:
```
30 seconds tak DNS cache valid rahega
```

<br>
<br>

### Step 11 — Resolver User ko Response deta hai

Ab recursive resolver IP address user ko bhej deta hai.

Flow:
```
Traffic Manager
      ↓
Recursive Resolver
      ↓
User Computer
```

<br>
<br>

### Step 12 — Browser Server se Connect karta hai

Ab browser direct connection banata hai.

Example:
```
User → Asia Server
```

Protocol:
```
HTTPS
```

Connection establish hone ke baad:
```
Website load ho jati hai
```

<br>
<br>

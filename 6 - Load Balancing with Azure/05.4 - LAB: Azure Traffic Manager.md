# LAB: Azure Traffic Manager

Aaj hum ek complete hands-on lab karte hain jisme tum Azure Traffic Manager use karke 2 VMs (West Europe + East US) ke beech global routing implement karoge.

Isme hum ek traffic manager create karenge, uske backend mein 2 linux vms hongi usne nginx server chal rha hoga jo simple html page render karega.

<br>
<br>

### Architecture (Jo hum banayenge)

Final architecture kuch aisa hoga:
```
User Browser
      |
DNS Query
      |
globaldemo.trafficmanager.net
      |
Azure Traffic Manager
      |
-------------------------------
|                             |
VM - West Europe        VM - East US
(Web Server)            (Web Server)
```

<br>

**Goal**:
- Agar user Europe se aaye → West Europe VM.
- Agar US se aaye → East US VM.
- Agar ek VM down ho jaye → automatic failover.

<br>
<br>

### Step 1 — Resource Group Create karo

Azure Portal → Resource Groups

Create karo:
```
Resource Group Name: tm-lab-rg
Region: East US
```

<br>
<br>

### Step 2 — West Europe VM Create karo

Azure Portal → Virtual Machines → Create

Configuration:
```
Name: vm-west-europe
Region: West Europe
Image: Ubuntu 22.04
Size: Standard B1s
Authentication: SSH / Password
Public IP: Enable
```
Network settings:
```
Allow HTTP (port 80)
Allow SSH (port 22)
```

Create VM.

<br>
<br>

### Step 3 — East US VM Create karo

Same process repeat karo.

Configuration:
```
Name: vm-east-us
Region: East US
Image: Ubuntu 22.04
Size: Standard B1s
Public IP: Enable
```
Open ports:
```
80
22
```

Create VM.

<br>
<br>

### Step 4 — Dono VM par Web Server Install karo

SSH login karo.

Example:
```
ssh azureuser@VM_PUBLIC_IP
```

Install nginx:
```
sudo apt update
sudo apt install nginx -y
```

<br>
<br>

### Step 5 — Unique Web Page Create karo

West Europe VM par:
```
sudo nano /var/www/html/index.html
```
Content:
```
<h1>Hello from West Europe VM</h1>
```

East US VM par:
```
sudo nano /var/www/html/index.html
```
Content:
```
<h1>Hello from East US VM</h1>
```

Browser test:
```
http://WEST_VM_PUBLIC_IP
http://EAST_VM_PUBLIC_IP
```

Agar dono open ho rahe hain:
```
Lab ka compute layer ready hai.
```

<br>
<br>

### Step 6 — Traffic Manager Profile Create karo

Azure Portal → Traffic Manager Profiles

Jab tum azure par traffic manager search karoge to wha tumko traffic manager profile option dikhta hai.

Create new.

Configuration:
```
Name: global-tm-demo
Routing method: Performance
Resource Group: tm-lab-rg
```

DNS name automatically banega:
```
global-tm-demo.trafficmanager.net
```
Create profile.

<br>
<br>

### Step 7 — Endpoints Add karo

Ab Traffic Manager ko batana hai ki kaunse servers available hain.

Traffic Manager → Endpoints → Add

Endpoint 1
```
Name: west-europe-endpoint
Type: External Endpoint
Target: WEST_VM_PUBLIC_IP
Location: West Europe
```

Endpoint 2
```
Name: east-us-endpoint
Type: External Endpoint
Target: EAST_VM_PUBLIC_IP
Location: East US
```

Save.

Ab Traffic Manager ko pata hai:
```
2 backend servers available hain
```

<br>
<br>

### Step 8 — Health Check Configure karo

Traffic Manager automatically health check karta hai.

Settings:
```
Protocol: HTTP
Port: 80
Path: /
Interval: 30 sec
```

Traffic Manager har 30 seconds mein check karega:
```
http://VM_IP/
```
Agar response:
```
200 OK
```
Endpoint healthy.

<br>
<br>

### Step 9 — Traffic Manager DNS Test

Ab browser mein open karo:
```
http://global-tm-demo.trafficmanager.net
```
Traffic Manager decide karega:
```
kaunsa endpoint fastest hai
```
Result:
```
Hello from West Europe VM
```
ya
```
Hello from East US VM
```

<br>
<br>

### Step 10 — Real Routing Test

Testing tools:
```
curl
```

Example:
```
curl global-tm-demo.trafficmanager.net
```

Response:
```

```

<br>
<br>

### Step 11 — Failover Test

Ab West Europe VM stop karo.

Azure Portal:
```
vm-west-europe → Stop
```
Wait:
```
1–2 minutes
```
Traffic Manager health probe detect karega:
```
West Europe → Unhealthy
```
Ab browser refresh karo:
```
global-tm-demo.trafficmanager.net
```
Result:
```
Hello from East US VM
```
Ye hai automatic failover.

<br>
<br>

### Real Production Architecture

Production environment mein architecture kuch aisa hota hai:
```
Users
   |
DNS
   |
Traffic Manager
   |
--------------------------------
|                              |
West Europe Region           East US Region
   |                              |
Application Gateway         Application Gateway
   |                              |
Load Balancer                Load Balancer
   |                              |
VM Scale Set                 VM Scale Set
```

Traffic Manager ka role:
```
Global traffic routing
```
Regional load balancing karta hai:
```
Application Gateway / Load Balancer
```

<br>
<br>

### Lab Learning Outcomes

Is project se tumne sikha:

**Global Load Balancing**:

User ko nearest region milta hai.

**High Availability**:

Ek region fail ho jaye to:
```
Traffic automatically dusre region par shift
```

**DNS Based Routing**:

Traffic Manager request route nahi karta.

Ye sirf DNS response deta hai.
